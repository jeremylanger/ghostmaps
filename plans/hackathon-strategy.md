## Judging Process

### AI Agent Judges + Human Judges
- **March 18:** Agent judges provide mid-hackathon feedback
- **March 22:** Final evaluation by agents + humans
- Open Track prize = "the synthesis of all values across all agent judges"
- Sponsors (Celo, Venice, etc.) have their own agent judges for bounties

### What AI Judges Means for Us
- Code quality, documentation, and machine-readable artifacts matter enormously
- A working demo that an AI agent can interact with beats a flashy video
- On-chain artifacts are verifiable by agents
- Clear README, API docs, well-structured code
- No dark patterns, no hidden state — agents can inspect everything

## Winning Strategy

### What makes us strong
- Solves a real, massive problem ($7.1B+ in Google privacy penalties)
- Deep Venice integration (6 touchpoints, not a wrapper)
- On-chain with real UX (CDP embedded wallets, gas sponsorship, EAS)
- Technically ambitious but achievable
- Privacy as architecture, not marketing
- Nobody else is building this (confirmed by competitive research)

### What to prioritize for judges
1. **Working end-to-end demo** that AI agents can actually use
2. **Excellent documentation** (README, API docs, architecture diagram)
3. **On-chain proof** (EAS attestations verifiable on basescan)
4. **Clear narrative:** "Google Maps tracks you every 2 minutes and has paid $7.1B+ in fines. We built the alternative."
5. **Privacy comparison page** — visual, factual, compelling

### Risks
- 9 days is tight for this scope
- If core features are buggy, polish won't save us
- AI judges might not be able to test navigation (no physical movement)
- Other teams might have simpler, more polished entries

### Mitigation
- Cut list is clear (navigation polish first, core search last)
- Day 9 dedicated to documentation for AI judges
- Seed demo reviews so judges see full experience
- Focus on reliability over features — working basics > broken advanced features
